修复MLX5启用SRIOV时，客户机报hw csum failure，导致掉线

原因

Proxmox默认启用iommu.passthrough=0,日志表现为iommu: Default domain type: Translated.

此模式与MLX5驱动存在兼容性问题，会导致分配VF的客户机出现(NULL net_device): hw csum failure报错。在OpenWrt上，此错误并不会导致立即离线，而是在第二次出现(约10d)时离线.

如何修复

在/etc/default/grub的GRUB_CMDLINE_LINUX_DEFAULT中加入iommu=pt

随后执行update-grub，重启即可。

此参数将启用Bypass the IOMMU for DMA,日志表现为iommu: Default domain type: Passthrough (set via kernel command line).

补充

此问题仅在ConnectX4LX(MCX4421A-ACQ_Ax)测试，其他使用MLX5的NIC尚不清楚是否存在此问题.
